## Actualizaciones Recientes (v1.4.0)

### Sistema de Medallas y Gamificación
- **Arquitectura Local-First**: Implementación de BadgeAssets con renderizado instantáneo (0ms latency)
- **5 Medallas del Sistema**: MVP, Boss, GPT, Fósil y Fantasma con iconografía Material Design
- **UX Toggle Instantáneo**: Sistema metralleta sin confirmaciones (tap gris = otorgar, tap color = quitar)
- **Background Sync**: Mapeo icon_key → UUID con caché en memoria
- **Optimistic Updates**: Feedback visual inmediato con rollback automático en errores
- **Clean Architecture**: Separación completa Domain/Data/Presentation layers

### Bug Fixes Críticos (v1.3.0)
- **Persistencia de sesión offline corregida**: Sesión mantenida al matar/reabrir la aplicación
- **Validación de tokens de Supabase**: Detección y recreación automática de sesiones expiradas
- **Eliminación de doble bucle O(n²)**: Cálculo de promedio optimizado a O(1) en campus_screen

### Optimizaciones de Rendimiento
- **Cache de avatares**: CachedNetworkImage reduce tráfico de red en 90%
- **Theme.of(context) optimizado**: Pasado como parámetro en métodos privados (83% menos lookups)
- **ScaffoldMessenger seguro**: Capturas antes de operaciones async para prevenir crashes
- **Mapeo JSON corregido**: Backend icon_key mapeado correctamente con JOIN queries

## Descripción General

UNSAP proporciona una interfaz moderna y optimizada para el seguimiento de calificaciones, métricas de rendimiento académico, red social estudiantil y datos del perfil estudiantil.

<table>
<tr>
<td width="50%">

**Características Principales**

- Sincronización vía web scraping del portal institucional
- Persistencia local con Hive para acceso offline
- Visualización de estadísticas con gráficos interactivos
- Red social estudiantil (Campus) con Supabase
- Sistema de notificaciones en tiempo real (Realtime)
- Sistema de badges por niveles de verificación
- Sistema de medallas gamificado con arquitectura local-first
- Tema oscuro/claro con persistencia
- Simulador de notas interactivo

</td>
<td width="50%">

**Optimizaciones de Rendimiento**

- Parsing de HTML en Isolates (no bloquea UI)
- Memoización de cálculos O(1) en ViewModels
- Lazy Loading diferido para pantallas pesadas
- Staggered loading de gráficos fl_chart
- ValueNotifier para rebuilds granulares
- ColoredBox anti-flash en transiciones
- CachedNetworkImage para avatares (caché L1/L2)
- Theme.of(context) pasado como parámetro
- TimeAgoManager singleton para Timers compartidos
- RepaintBoundary en widgets de lista

</td>
</tr>
</table>

## Instalación

1. Clonar el repositorio:

```bash
git clone https://github.com/JOSETRA44/unsap.git
cd unsap
```

2. Configurar variables de entorno:

```bash
cp .env.example .env
# Editar .env con las credenciales de Supabase
```

3. Instalar dependencias:

```bash
flutter pub get
```

4. Ejecutar la aplicación:

```bash
flutter run
```

## Arquitectura

El proyecto implementa una arquitectura **MVVM** (Model-View-ViewModel) con separación clara de responsabilidades:

```
┌─────────────────────────────────────────────────────────────┐
│                    Presentation Layer                       │
│              (Screens, Widgets, UI Components)              │
└─────────────────────────┬───────────────────────────────────┘
                          │
                          ▼
┌─────────────────────────────────────────────────────────────┐
│                     ViewModel Layer                         │
│         (State Management, Business Logic, Memoization)     │
└─────────────────────────┬───────────────────────────────────┘
                          │
                          ▼
┌─────────────────────────────────────────────────────────────┐
│                       Data Layer                            │
│              (Models, DataSources, Repositories)            │
└─────────────────────────┬───────────────────────────────────┘
                          │
                          ▼
┌─────────────────────────────────────────────────────────────┐
│                       Core Layer                            │
│           (Network, Storage, Theme, Services)               │
└─────────────────────────────────────────────────────────────┘
```

**Patrones Implementados:**

| Patrón | Implementación |
|--------|----------------|
| State Management | Provider con ChangeNotifier |
| Clean Architecture | Domain/Data/Presentation layers |
| Local-First Architecture | BadgeAssets con background sync |
| Memoization | Cálculos cacheados O(1) en ViewModels |
| Lazy Loading | Construcción diferida de pantallas pesadas |
| Keep Alive | AutomaticKeepAliveClientMixin en navegación |
| Isolates | Parsing de HTML en hilos separados |
| Repository Pattern | Abstracción de fuentes de datos (Supabase/Local) |
| UseCase Pattern | Lógica de negocio aislada en Domain layer |
| Singleton | Servicios centralizados (UserProfile, Identity, Notification) |
| Disposed Guard | Prevención de crashes por uso después de dispose() |
| Optimistic Updates | Feedback visual inmediato con rollback en errores |

- Cálculo de nota máxima alcanzable

  - Nivel 2+: Badge dorado (delegados/admins)
- Sistema de medallas gamificado:
  - 5 Medallas: MVP, Boss, GPT, Fósil, Fantasma
  - Arquitectura local-first (renderizado 0ms)
  - Toggle instantáneo sin confirmaciones
  - Background sync con mapeo icon_key → UUID
  - Optimistic updates con rollback automático
- Creación de posts con texto

- Velocímetro de promedio global (fl_chart)
- Gráfico de barras por curso
- Tarjetas de resumen (mejor/peor curso, cursos en riesgo)

### Ajustes

- Información personal (CUI, Email vinculado, Programa)

- Recuperación de cuenta: login con credenciales existentes
- Cambio de contraseña para cuentas vinculadas
- Confirmación de email con Deep Links
- Recuperación de contraseña por email

- Servicio de identidad centralizado

## Sistema de Temas

UNSAP incluye un sistema completo de temas con soporte para modo claro, oscuro y automático (según sistema).

### Paleta de Colores

| Modo | Background | Surface | Accent |
|------|------------|---------|--------|
| Light | `#F5F5F5` | `#FFFFFF` | `#8B0000` (Maroon) |
| Dark | `#121212` | `#1E1E1E` | `#FF6B6B` (Coral) |

### Arquitectura del Sistema de Temas

```dart
// lib/core/theme/app_theme.dart
class AppTheme {
  static ThemeData get light => ThemeData(...);
  static ThemeData get dark => ThemeData(...);
}

// lib/core/theme/theme_manager.dart
class ThemeManager extends ChangeNotifier {
  ThemeMode _themeMode = ThemeMode.system;
  // Persistencia con Hive
}
```

### Uso en Widgets

```dart
final theme = Theme.of(context);
final isDark = theme.brightness == Brightness.dark;
final primaryColor = isDark ? AppTheme.darkAccent : AppTheme.maroon;

Container(
  color: theme.cardTheme.color,
  child: Text('Texto', style: theme.textTheme.bodyLarge),
)
```

## Configuración

### Variables de Entorno

Crear archivo `.env` en la raíz del proyecto:

```env
SUPABASE_URL=https://tu-proyecto.supabase.co
SUPABASE_ANON_KEY=tu-anon-key
```

### Colores Institucionales

Los colores están centralizados en `lib/core/theme/app_theme.dart`:

```dart
class AppTheme {
  // Institucional
  static const Color maroon = Color(0xFF8B0000);
  
  // Dark Theme
  static const Color darkBackground = Color(0xFF121212);
  static const Color darkSurface = Color(0xFF1E1E1E);
  static const Color darkAccent = Color(0xFFFF6B6B);
}
```

### Splash Screen Nativo

Configurado en `android/app/src/main/res/values/styles.xml` con el color institucional para una transición imperceptible entre la capa nativa y Flutter.
